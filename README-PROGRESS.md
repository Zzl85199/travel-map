## 這次的更新（登入方式 / 中文化 / Q版美化 / 搜尋修正）

- **登入改用「重新導向」(redirect) 模式**，不再用彈出視窗(popup)。因為你確認過 Google Cloud 的 API 金鑰限制已經排除，登入依然失敗，最有可能是瀏覽器的第三方 Cookie／隱私設定卡住了 popup 流程用的隱藏 iframe。redirect 模式會直接整頁導到 Google 登入頁再導回來，不依賴那個 iframe，環境相容性通常更好。
  - **如果這次還是不行**：麻煩告訴我登入後網址列有沒有跳轉到 `accounts.google.com` 又跳回來、還是完全沒反應；如果有跳轉但跳回來後還是顯示「尚未登入」，问题很可能在 Firebase Console → Authentication → Settings → Authorized domains 有沒有列出 `johnny-journey.web.app`（這個通常會自動加入，但麻煩去確認一下）。
- **所有錯誤訊息中文化**：以前 Firebase 回傳的英文錯誤（像 network-request-failed 那一長串）現在會轉換成中文說明，其他像儲存失敗/刪除失敗/新增協作者失敗等訊息也一併處理。
- **搜尋功能加了看得到的回饋**：搜尋框下方會顯示「找到 X 個地點」，同時地圖會自動縮放/移動到符合條件的地點，這樣就能明顯看到搜尋真的有作用（之前只有標記消失/出現，資料量少時不容易注意到）。
- **2D 地圖 Q 版美化**：
  - 圖釘改成圓潤色塊＋分類 emoji（景點🏛️／美食🍜／住宿🏨／交通🚌／購物🛍️／其他⭐），會輕輕上下浮動
  - 底圖加了飽和度／對比濾鏡，顏色更活潑不呆板
  - 彈出卡片改成更圓的圓角、按鈕變成膠囊形狀

## 這次的更新（地圖循環 / 定位工具 / 設定值分離）

- **地圖左右可以無限循環滾動**（`worldCopyJump: true`），上下因為地球極點在平面地圖上是「邊緣」不是「循環點」沒辦法真的循環，改成拖到接近邊緣就自動彈回來，不會再看到空白區域。
- **修正新增地點位置跑掉的 bug**：因為地圖現在左右無限循環，`地圖中心`的經度數值有時會超出 -180~180 的正常範圍（例如變成 490 度），直接存進資料庫的話下次讀取會顯示在錯誤的地方（Toronto 跑到亞洲那次就是這個原因）。現在存檔前都會先把座標「歸零」到正常範圍。
- **新增地點表單裡加了「🔍 定位」快速搜尋**：輸入城市或地址名稱（例如「Toronto, Canada」），會用 OpenStreetMap 的免費地理編碼服務直接跳到那個位置，不用自己在地圖上找。
- **編輯地點時現在也會顯示可拖曳的定位標記**：像 Toronto 那種位置跑掉的舊資料，直接打開編輯、把橘色標記拖到正確位置、按儲存就修好了，不用刪除重建。
- **設定值搬到獨立的 `public/config.js`**：Firebase 和 Cloudinary 的設定值不再寫在 `index.html` 裡面，改成獨立檔案讀取。這個檔案已經幫你填好你目前的真實設定值，加進了 `.gitignore`（推 GitHub 不會帶到這個檔案，你的金鑰值本來就不是機密但這樣比較乾淨）。之後我幫你調整功能只會動 `index.html`，不會動這個檔案，你不用每次都重貼一次。`public/config.example.js` 是空白範本，換電腦或找新協作者時可以參考它重建 `config.js`。

## 還沒做 / 你要自己做的事








1. **在 `index.html` 填入你自己的 Firebase 設定值**（`firebaseConfig` 那一段），
   跟你原本的做法一樣，這次沒有改這部分的結構。
2. **申請 Cloudinary 免費帳號並設定**（取代 Firebase Storage，不用信用卡）：
   - 前往 https://cloudinary.com 註冊免費帳號
   - Dashboard 首頁會看到 **Cloud name**，填到 `index.html` 的 `CLOUDINARY_CLOUD_NAME`
   - 到 Settings → Upload → Upload presets → **Add upload preset**
     - Signing Mode 選 **Unsigned**（很重要，前端才能直接上傳不需要密鑰）
     - 存檔後把 preset 名稱填到 `index.html` 的 `CLOUDINARY_UPLOAD_PRESET`
3. **手動建立第一個 owner**：
   規則設計是「只有 owner 能新增/刪除協作者」，所以你要自己到 Firebase Console
   的 Firestore，手動新增一筆文件：
   - collection: `editors`
   - 文件 ID：你自己的 Google 帳號 Email
   - 欄位：`role` = `"owner"`（字串）
4. 部署規則與網站：
   ```bash
   firebase deploy --only firestore:rules,hosting
   ```
5. 建議測試流程：
   - 用你自己帳號登入 → 確認地球會旋轉、點光點能飛進城市地圖
   - 新增一個地點、上傳一張照片 → 確認地圖卡片有照片顯示出來（可以去 Cloudinary
     Dashboard 的 Media Library 確認檔案真的傳上去了）
   - 到管理面板新增一條路線，把 2～3 個地點設定同一個 `routeId`、不同 `order`
     → 到路線分頁點「調整站序」拖曳測試，確認地球/地圖上的路線顏色跟著變
   - 邀請一位朋友的 Email 進協作者名單，請他登入測試新增/編輯
